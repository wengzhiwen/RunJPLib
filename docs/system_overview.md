# 系统概述

RunJPLib 是一个基于 Flask 的 Web 应用程序，为用户提供日本大学的招生信息和相关博客文章。它具有强大的后台管理功能，利用 AI 技术简化内容创作和数据处理。

## 主要功能

### 1. 大学信息库
- **核心功能**: 提供一个可搜索的大学信息数据库，包含每所大学的详细招生信息。
- **数据来源**: 信息由管理员通过后台上传 PDF 格式的招生简章，系统会自动处理并提取结构化数据。
- **用户界面**:
    - **首页**: 展示最新的大学信息、推荐博客和按分类（国公立/私立/地区）组织的大学快速索引。
    - **大学详情页**: 显示特定大学的招生简章原文（OCR 日文）、翻译（中文）、分析报告以及 PDF 文档。
    - **中文名支持**: 支持通过大学日文名或中文名进行查找，自动回退匹配。

### 2. 博客系统
- **功能**: 发布与日本留学、大学介绍相关的博客文章。
- **AI 辅助创作**: 管理员可以使用 AI 博客生成器，通过提供大学材料和提示词，快速创建高质量的博客内容。支持三种生成模式：材料扩展、对比分析、纯用户提示词。
- **Wiki 功能**: 博客内容中的大学名称会自动链接到相应的大学信息页面。
- **可见性控制**: AI 生成的博客默认为私密状态，需管理员审核后手动公开。
- **异步生成**: 博客生成任务在后台异步执行，避免超时。

### 3. AI 对话系统
- **智能问答**: 用户可以就特定大学的招生信息与 AI 进行对话。
- **混合搜索**: 对话系统采用向量搜索和关键词搜索相结合的混合策略，以确保查询结果的准确性，特别是对于专业术语。
- **隐私保护**: 系统采用基于浏览器会话的隐私保护机制，确保用户对话的私密性。
- **同义词理解**: 支持中日文同义词扩展，提高检索召回率。

### 4. 后台管理面板 (`/admin`)
一个功能强大的内部工具，供管理员管理网站内容和监控系统状态。
- **数据管理**:
    - **招生信息管理**: 管理所有大学的招生信息，包括增删改查。
    - **博客管理**: 管理所有博客文章，包括在线编辑和 AI 生成。
- **内容处理工具**:
    - **招生信息处理器**: 上传并处理 PDF 招生简章，自动提取信息。支持普通模式和批量模式（成本优化）。支持可选的参考 Markdown 校对补强。
    - **招生信息再生成**: 在大学编辑页面，可通过修改系统提示词重新生成分析报告。
    - **大学标签工具**: 利用 AI 批量为大学自动生成和更新标签（国立/私立/难关等）。
    - **AI 博客生成器**: 辅助管理员创建博客文章。
- **系统监控**:
    - **仪表盘**: 显示核心数据统计，如大学数量、博客数量、访问量、24h 对话数等。
    - **线程池监控**: 实时监控后台任务线程池的状态。
    - **聊天会话履历**: 查看所有用户的聊天会话记录。
    - **TOP30 访问**: 按 IP 去重查看不同时间窗口（24h/48h/7天）的 TOP30 访问页面。
    - **IP 地理位置分析**: 查看最近访客的地理位置信息。

## 技术栈
- **后端**: Flask + Gunicorn
- **数据库**: MongoDB (使用 GridFS 存储大型 PDF 文件) + ChromaDB (向量存储)
- **AI 集成**:
    - **内容生成与分析**: OpenAI GPT 系列（通过 openai-agents 框架）
    - **向量搜索**: LlamaIndex + ChromaDB
    - **工作流编排**: Buffalo Workflow
- **认证与安全**: Flask-JWT-Extended (JWT + HttpOnly Cookie) + CSRF 保护
- **前端**: HTML, CSS, JavaScript (Bootstrap 5.3)
- **实时通信**: Server-Sent Events (SSE) 用于后台状态推送
- **日志**: Python logging + 按日分割日志文件
