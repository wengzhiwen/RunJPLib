# 推荐阅读算法

为了在博客和首页向用户展示相关且及时的内容，RunJPLib 实现了一套带时间权重的推荐算法，并结合了缓存机制以优化性能。

## 算法逻辑

核心目标是让最新的博客文章有更高的曝光率，同时保证推荐内容的多样性。

### 1. 时间分组
算法将所有公开博客按发布日期分为三组：
- **最近 3 天**: 最高权重组。
- **最近 7 天**: 中等权重组。
- **更早**: 基础权重组。

### 2. 选择步骤 (以推荐 3 篇为例)
1.  **优先选择最新**: 从"最近 3 天"的博客中，随机选择最多 2 篇。
2.  **智能补足**: 如果上一步选不满，从"最近 7 天"中选择不重复的文章，补足 3 篇。
3.  **兜底选择**: 如果前两步仍不足 3 篇，从"更早"的博客中随机选择补足。

这种机制确保了只要有新文章，它们就极有可能被推荐，同时通过随机选择保证了一定的多样性。

## 性能优化：TTL 缓存

- **缓存策略**: 使用 `cachetools.TTLCache`（`maxsize=1, ttl=1800`，即 30 分钟有效期）。
- **工作流程**: 请求推荐内容时先查缓存；缓存未命中则执行算法并存入缓存。
- **缓存自动失效**: 管理员在后台创建或更新博客文章时，推荐缓存被自动清除。

## 技术实现

- **核心函数**: `routes/blog/views.py` 中的 `get_weighted_recommended_blogs_with_summary()`。
- **缓存定义**: `routes/blog/cache.py` 中的 `recommended_blogs_cache`。
- **缓存清除**: `routes/blog/cache.py` 中的 `clear_recommended_blogs_cache()`。
- **摘要生成**: 使用 `markdown` 库将 Markdown 转换为纯文本摘要。
- **应用场景**: 首页推荐、博客详情页推荐、404 页面推荐。
