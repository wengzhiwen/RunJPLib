# 线程池架构优化需求文档

## 概述

本文档描述了RunJPLib项目中线程池架构的优化需求，主要解决当前线程池命名混淆

## 优化需求

### 1. 线程池命名优化

#### 1.1 Analytics日志线程池 → 用户访问日志线程池

**当前命名**: "Analytics日志线程池"
**优化后命名**: "用户访问日志线程池"
**理由**: 
- 实际用途是记录用户页面访问日志
- 避免与数据分析功能混淆
- 命名更加直观明确


#### 1.2 博客更新线程池 → 博客HTML构建线程池

**当前命名**: "博客更新线程池"
**优化后命名**: "博客HTML构建线程池"
**理由**:
- 实际用途是博客HTML重建（Markdown转HTML）
- 与Admin线程池的博客保存/更新功能区分开
- 避免功能重复的误解

同时更新必要的方法命名和文档等。
