# AI 对话系统

RunJPLib 提供了一个智能 AI 对话系统，允许用户就特定大学的招生信息进行提问和互动。

## 核心特性

### 1. 混合搜索 (Hybrid Search)

对话系统采用向量搜索和关键词搜索相结合的混合策略：

- **向量搜索**: 通过 LlamaIndex + ChromaDB 进行语义匹配，理解上下文和含义。
- **关键词搜索**: 使用正则表达式进行精确术语匹配，确保专业术语不会遗漏。
- **智能合并**: 根据预设权重和精确匹配加成分数，合并两种搜索结果并重排序。
- **并行执行**: 两种搜索使用线程池并行执行，提升响应速度。

### 2. 隐私保护

- **浏览器会话隔离**: 基于 `sessionStorage` 的唯一会话 ID，确保同一网络下不同用户、普通/无痕模式、不同标签页完全隔离。
- **双重验证**: 恢复会话时优先使用浏览器会话 ID，旧会话回退使用 IP 地址验证。
- **即时释放**: 每次搜索完成后立即释放临时数据。

### 3. 同义词理解

- **查询扩展**: 加载 `wasei_kanji.csv` 同义词库，自动检测关键词并用同义词扩展查询（如"出願"与"报名"）。
- **LLM 查询分析**: 使用 LLM 分析用户查询，提取精确匹配和模糊匹配关键词，进一步提升检索质量。

### 4. 向量索引自动更新

- 系统在对话时自动检测向量索引是否与 MongoDB 中的最新数据一致。
- 如果检测到数据更新，自动重建该大学的向量索引，确保对话基于最新信息。

### 5. 内存与性能优化

- **实时监控**: 使用 `psutil` 监控内存使用。
- **自动清理**: 超过阈值时自动清理缓存和临时对象。
- **正则缓存**: 编译后的正则表达式缓存，避免重复编译。

## API 接口

对话系统通过 RESTful API (`/api/chat/{university_name}/`) 提供服务：

- `POST /create-session`: 创建或恢复安全会话
- `POST /send-message`: 发送消息，触发混合搜索和 AI 回答
- `GET /get-history`: 获取当前会话历史消息
- `POST /clear-session`: 清理会话历史（管理端专用）
- `DELETE /delete-session`: 删除会话（管理端专用）

### 管理端代理路由

管理端通过 `/admin/chat/api/<path:endpoint>` 代理路由访问相同的 API，携带 `X-Role-Token: 'admin'` 头部。

## 对话检索专项日志

所有检索操作记录到独立的按日分割日志文件 `log/retrieval_YYYYMMDD.log`，包含会话 ID、原始查询、扩展查询、搜索结果和 AI 回答等详细信息，便于分析搜索效果。
