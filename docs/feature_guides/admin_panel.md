# Admin 面板功能指南

## 访问与安全

- **访问地址**: `/admin`
- **认证**: 管理员在登录页面输入通过环境变量 `ACCESS_CODE` 配置的访问码。成功登录后，系统使用 HttpOnly Cookie 维持会话。`@admin_required` 装饰器区分页面请求（重定向登录页）和 API 请求（返回 JSON 错误）。

## 主要功能模块

### 1. 仪表盘 (`/admin`)
后台的欢迎页面，提供系统核心指标的快照。

- **数据一览**: 实时显示收录的学校总数、博客文章总数、24 小时独立访客数、对话 IP 数、对话数以及各主要页面的浏览量。
- **过期 Premium 学校**: 显示所有报名截止日期已过期的 Premium 大学列表，可直接跳转到招生信息生成页面并自动填充大学名称。
- **系统状态监控**: 实时展示三个核心后台线程池（用户访问日志、博客 HTML 构建、Admin 操作）的运行状态，包括活跃线程数、队列大小和负载情况。
- **实时数据推送**: 通过 SSE (Server-Sent Events) 每 30 秒推送一次统计数据更新。

### 2. 招生信息管理

- **招生信息一览 (`/admin/manage/universities`)**: 以表格形式展示所有大学的招生信息，支持基于标签的筛选（最多选 2 个标签，AND 交集），显示大学的日文名、中文名、Premium 状态和报名截止日期。管理员可进行编辑或删除操作。

- **招生信息编辑 (`/admin/edit_university/<id>`)**: 在线编辑大学的中文名、截止日期等基础信息，以及三份 Markdown 内容（OCR 原文、中文翻译、分析报告）。包含"再生成分析报告"按钮。

- **招生信息生成 (`/admin/pdf/processor`)**: 上传大学的 PDF 招生简章，可选附带参考 Markdown 进行校对补强，选择处理模式（普通/批量），启动自动化的信息处理流程。

- **任务列表 (`/admin/pdf/tasks`)**: 查看所有处理任务的进度、状态和日志。支持从任意步骤重启失败或中断的任务。手动刷新（已移除自动轮询）。

- **大学标签工具 (`/admin/university-tagger`)**: 利用 LLM 批量为所有大学自动生成和更新标签（国立/私立/难关/女子大等），作为后台异步任务执行，支持实时查看进度和结果。

### 3. 博客管理
- **Blog 一览 (`/admin/manage/blogs`)**: 管理所有博客文章。支持编辑、删除、可见性切换。
- **AI 博客生成器 (`/admin/blog/create`)**: 支持三种模式（材料扩展、对比分析、纯用户提示词），异步生成。参见 [博客管理文档](blog_management.md)。

### 4. 分析与洞察
- **独立 IP (24h) (`/admin/analytics/unique_ips`)**: 查看过去 24 小时内访问过网站的独立 IP 列表，利用 GeoIP 显示地理位置信息。
- **TOP30 访问 (`/admin/analytics/top30`)**: 按 IP 去重查看不同时间窗口（24h/48h/7 天）的 TOP30 访问页面，支持分别查看 Blog 与招生信息。

### 5. 聊天管理
- **AI 对话测试 (`/admin/chat`)**: 供管理员测试和调试 AI 对话功能。可选择任意大学直接与招生信息进行对话，支持清理历史和删除会话。
- **聊天会话履历 (`/admin/chat-logs`)**: 查看所有用户的聊天会话列表（按最后活动时间逆序），可点击进入单个会话详情页查看完整对话内容。仪表盘显示 24 小时对话 IP 数和总对话数。

## 标签筛选 API

- `GET /admin/api/university-tags`: 返回所有标签及其包含的招生信息数量
- `GET /admin/api/universities?tags=国立,理系`: 最多支持两个标签，逗号分隔，AND 交集查询
