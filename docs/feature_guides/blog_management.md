# 博客管理与 AI 生成

RunJPLib 提供了一套完整的博客管理功能，并集成了强大的 AI 辅助创作工具，帮助管理员高效地生成和管理内容。

## 博客管理

管理员可以通过后台的"Blog 一览"页面 (`/admin/manage/blogs`) 对所有博客文章进行管理。

- **查看与编辑**: 可以查看所有已发布的博客，并直接在线编辑任何一篇文章的标题和 Markdown 原文。
- **删除**: 可以删除不再需要的博客文章。
- **新建**: 点击"新建博客"按钮，即可进入 AI 博客生成器页面。也支持直接输入标题和内容手动创建。

## AI 博客生成器

管理后台的核心内容创作工具，允许管理员通过 AI 辅助以多种模式创建博客文章。

### 生成模式

1.  **材料扩展模式**: 基于一所或多所大学的招生简章原文，结合用户的具体要求进行扩展创作。支持 Web Search 补充材料。
2.  **对比分析模式**: 生成多所大学的横向对比或综合推荐类文章。系统会先智能缩减每所大学的原文材料（提取核心要点），并纳入分析报告，然后交给 AI 进行对比。
3.  **纯用户提示词模式**: 完全脱离现有的大学材料，仅根据管理员输入的提示词或主题进行创作。

### 内容生成架构

使用 `openai-agents` 框架，采用多步 Agent 协作：
- **内容生成 Agent**: 根据模式和参数生成初稿
- **Formatter Agent**: 对生成内容进行格式化，输出 JSON 结构化结果
- **内容缩减器**: 在对比模式下，智能缩减长文本材料以控制 token 消耗

### 使用流程 (异步)

1.  配置生成参数并提交。系统立即响应，后台线程池执行生成。
2.  生成完成后，新文章默认为"私密"状态，出现在"Blog 一览"页面。
3.  管理员审核、修改后切换为"公开"发布。

### 429 错误处理

当遇到 OpenAI API 速率限制（429 错误）时，系统自动使用基础分析报告 (`report_md`) 替代长文本原始内容，确保生成功能可用。

## 可见性控制与发布

- **`is_public` 字段**: 每篇博客都有公开/私密状态。
- **默认状态**: AI 生成的博客默认为 `私密`，手动创建的博客默认为 `公开`。
- **前端显示**: 所有公开部分（博客列表、首页推荐、站点地图等）只显示"公开"文章。

## 快速再生成

在编辑 AI 生成的博客时，页面上会出现"再生成"按钮。点击后自动在新标签页打开"新建博客"页面，并预填充上一次生成时使用的所有参数，方便快速迭代。

## 自动 Wiki 功能

在博客文章被保存或更新时，系统自动识别内容中出现的大学名称（支持中文和日文），并为其添加指向本站对应大学信息页面的超链接。采用最长匹配原则，避免破坏已有链接和代码块。详见 [Wiki 功能文档](blog_wiki.md)。
